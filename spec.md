# 《心动日常》情侣专属系统 - 项目大纲

## 一、项目概述

### 1.1 项目名称
《心动日常》情侣专属双端系统

### 1.2 项目背景
为情侣打造专属的私密互动系统，融合恋爱纪念、生活贴心管理、日常互动、正向恋爱反馈四大核心能力，替代零散的APP/表格，打造专属于两个人的恋爱空间，兼顾浪漫仪式感与日常实用性。

### 1.3 核心目标
1. 实现情侣恋爱全周期的纪念留存，打造专属时光档案
2. 解决女生日常痛点，提供贴心的生活管理能力，体现心意
3. 提供轻量化的双人互动能力，提升日常恋爱粘性
4. 通过极简的积分/等级体系，形成恋爱正向反馈，无开发冗余
5. 一套代码实现多端覆盖，保证移动端优先的使用体验

### 1.4 项目范围
- 用户注册/登录、情侣双向绑定体系
- 时光档案馆模块（时光线、纪念日、心愿清单、悄悄话信箱）
- 专属生活管家模块（生理期管理、饮食偏好档案、全能提醒、好物收纳）
- 双人互动空间站模块（双人打卡、积分商城、情绪树洞、情侣账本）
- 轻量化恋爱养成模块（心动分等级、成就体系、福利兑换）
- 系统定时提醒、权限控制、多端适配基础能力

## 二、技术架构

### 2.1 整体架构
采用前后端分离架构，分为4层：
1. **表现层**：uniapp前端项目，负责页面渲染、用户交互、数据请求
2. **接口层**：FastAPI后端服务，提供RESTful标准接口
3. **业务层**：后端服务核心业务模块
4. **数据层**：MySQL数据库，通过SQLAlchemy ORM交互

### 2.2 前端技术栈
| 技术 | 说明 |
|------|------|
| uniapp | 跨端开发框架，编译为H5、微信小程序、APP |
| Vue3 | 使用setup语法糖，组合式API |
| Vant UI | 统一UI组件库 |
| Pinia | 全局状态管理 |
| SCSS | 样式预处理器 |

### 2.3 后端技术栈
| 技术 | 说明 |
|------|------|
| Python 3.10+ | 开发语言 |
| FastAPI | Web框架 |
| SQLAlchemy | ORM数据库映射 |
| APScheduler | 定时任务调度 |
| MySQL 8.0 | 数据库 |
| JWT | 鉴权方案 |

## 三、功能模块设计

### 3.1 用户与情侣绑定模块（基础核心）

#### 3.1.1 用户注册
- 输入：账号（3-32位，唯一）、密码（6-32位）、昵称（2-32位）
- 输出：注册成功/失败提示
- 规则：自动生成6位大写字母+数字邀请码，密码bcrypt加密存储

#### 3.1.2 用户登录
- 输入：账号、密码
- 输出：JWT Token（有效期30天）
- 规则：校验账号密码，成功返回Token

#### 3.1.3 情侣绑定
- 输入：对方6位邀请码
- 输出：绑定成功/失败提示
- 规则：校验双方未绑定状态，完成双向绑定

#### 3.1.4 用户信息管理
- 可编辑：昵称、头像、密码
- 展示：个人信息、邀请码、情侣信息

### 3.2 时光档案馆模块（浪漫核心）

#### 3.2.1 时光线管理
- 核心字段：标题、内容、事件时间、多图、发布人、发布时间
- 权限：仅发布者可编辑/删除，情侣可查看
- 展示：瀑布流卡片布局，支持下拉刷新、上拉加载

#### 3.2.2 纪念日管理
- 核心字段：标题、目标日期、是否循环、提醒天数、备注
- 权限：个人纪念日/情侣纪念日，情侣双方均可查看/编辑
- 功能：自动计算倒计时，提前提醒

#### 3.2.3 心愿清单管理
- 核心字段：内容、状态、完成时间、备注、图片
- 权限：个人心愿/情侣心愿，情侣双方均可操作
- 功能：标记完成自动同步时光线

#### 3.2.4 悄悄话信箱
- 核心字段：发送人、接收人、内容、时间、已读状态、定时发送
- 权限：仅绑定情侣可互发
- 功能：定时发送、已读状态标记

### 3.3 专属生活管家模块（实用核心）

#### 3.3.1 生理期管理
- 核心字段：开始日期、周期天数、持续天数、备注、身体状态
- 权限：仅本人可编辑，情侣仅可查看预测信息
- 功能：自动预测下次生理期、排卵期，提前3天提醒

#### 3.3.2 饮食偏好档案
- 核心字段：喜欢的食物、忌口、过敏信息、口味偏好、外卖地址
- 权限：仅本人可编辑，情侣仅可查看
- 展示：表单式布局，分模块展示

#### 3.3.3 全能提醒中心
- 核心字段：标题、截止日期、提醒时间、备注、状态
- 权限：个人待办/情侣待办，情侣双方均可操作
- 功能：自动推送提醒

#### 3.3.4 好物收纳管理
- 核心字段：物品名称、品牌、型号、保质期、购买时间、开封时间
- 权限：仅本人可编辑，情侣可查看
- 功能：临期提醒，按分类管理

### 3.4 双人互动空间站模块（粘性核心）

#### 3.4.1 双人打卡系统
- 核心字段：打卡项目名称、心动分、创建人、是否共同打卡
- 权限：情侣双方均可创建、打卡、查看
- 功能：每日打卡获心动分，连续打卡奖励

#### 3.4.2 积分福利兑换池
- 核心字段：福利名称、所需心动分、兑换规则、是否可重复
- 权限：情侣双方均可新增、兑换、查看
- 功能：积分兑换、自动核销

#### 3.4.3 情绪树洞
- 核心字段：情绪类型、内容、发布时间、是否同步情侣
- 权限：可选择是否同步给情侣
- 功能：负面情绪自动暖心回复，同步提醒情侣

#### 3.4.4 极简情侣账本
- 核心字段：金额、类型、时间、支付人、备注、是否AA
- 权限：情侣双方均可新增，仅创建者可编辑
- 功能：月度统计、AA自动计算

### 3.5 轻量化恋爱养成模块（极简实现）

## 五、权限与安全设计

### 5.1 数据权限规则
1. **私密数据**：仅用户本人可查看/编辑（如个人饮食偏好、生理期记录）
2. **共享数据**：用户本人和绑定的情侣均可查看，仅发布者可编辑/删除（如时光线、心愿清单）
3. **双向数据**：双方均可操作，数据双向同步（如情侣账本、悄悄话、打卡项目）

### 5.2 安全要求
1. 密码必须bcrypt加密存储
2. 所有接口必须JWT鉴权
3. 所有情侣共享数据必须权限校验
4. 入参必须校验，防止SQL注入、XSS攻击
